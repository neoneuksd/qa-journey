# Manual QA Basics

## QA
QA is a person who checks product quality.

## Bug
A bug is when Actual Result does not match Expected Result.

## Requirement
A requirement is a description of how the product should work.

## Expected Result
Expected Result is what should have happened.

## Actual Result
Actual Result is what actually happened.

## Bug report
A bug report is a document that describes a found issue and how to reproduce it.

## Test case
A test case is a detailed instruction that describes what and how to test.

## Checklist
A checklist is a short list of what needs to be checked.

## Smoke testing
Smoke testing is a quick check that the product works at all.

## Sanity testing
Sanity testing is a check of a specific part of the product after a fix.

## Regression testing
Regression testing is a check that old functionality still works after changes.
